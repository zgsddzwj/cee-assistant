# 高考志愿填报系统 - 数据采集层设计方案

## 概述

本设计文档详细规划了高考志愿填报 AI 应用的数据采集层架构，采用混合采集模式确保数据的完整性、准确性和持续性。

## 一、数据架构

### 1.1 基础数据（年度更新）

#### 高考分数数据
- **内容**：各省市文理科批次线（近5年）、一分一段表数据
- **数据项**：省份、年份、科类、批次、分数线值、对应位次范围
- **更新频次**：高考成绩公布后立即同步（每年1次）
- **质量要求**：准确率100%，需多重验证

#### 院校信息
- **内容**：院校基本信息（985/211/双一流标识）、地理位置、属性
- **数据项**：院校代码、院校名称、办学性质、所在城市、隶属关系
- **更新频次**：长期稳定，年度核查（每年1次）

#### 专业数据
- **内容**：专业代码、名称、学科门类、培养目标概述
- **数据项**：专业代码、专业名称、学科门类、学位类型
- **更新频次**：年度专业目录发布后更新（每年2次）

### 1.2 扩展数据（长期积累）

#### 就业质量数据
- **内容**：就业率、平均薪资、行业分布
- **更新频次**：年度就业报告发布后同步（每年1-2次）

#### 学科评估数据
- **内容**：教育部学科排名、ESI排名
- **更新频次**：评估周期同步（每2-3年）

#### 行业发展数据
- **内容**：行业需求趋势、岗位数量变化
- **更新频次**：季度跟踪（每季度1次）

### 2.2 组件职责

| 组件 | 职责 | 技术栈 |
|------|------|--------|
| 数据调度中心 | 任务分发、监控报警、配置管理 | Python + APScheduler |
| 微爬虫集群 | 网页抓取、数据提取、本地存储 | Scrapy + Selenium + BeautifulSoup |
| API接口服务 | 对接政府开放平台、第三方数据接口 | Python + Requests + API Gateway |
| 数据清洗管道 | 格式校验、逻辑校验、数据去重 | Pandas + Great Expectations |
| 分级存储 | 热数据加速访问、核心数据持久化、冷数据归档 | Redis + PostgreSQL + MinIO |

## 三、微爬虫集群详细设计

### 3.1 考试院爬虫

#### 目标网站
- 31个省级教育考试院官网
- 阳光高考信息平台

#### 采集内容
- 历年各批次录取分数线
- 一分一段表
- 各院校录取最低分、最高分、平均分

#### 技术方案
- 使用 Selenium 处理动态加载
- 自适应解析不同网站结构
- IP轮换和请求频率控制
- 断点续爬和数据更新检测

#### 运行机制
- 数据采集窗口：6月-8月
- 每日检查新发布数据
- 异常情况自动重试

### 3.2 院校官网爬虫

#### 目标网站
- 全国各类高等院校招生网
- 各院校专业介绍页面

#### 采集内容
- 年度招生计划
- 专业介绍和课程设置
- 录取规则和特殊要求
- 就业质量报告

#### 技术方案
- BeautifulSoup + lxml 解析HTML
- 正则表达式匹配关键信息
- 多线程并发抓取
- 数据版本对比检测

#### 运行机制
- 分批次抓取（985 → 211 → 普通本科 → 高职高专）
- 招生季集中抓取（2-4月，10-12月）
- 增量更新检测机制

### 3.3 第三方数据整合

#### 数据源
- 教育部门户网站
- 国家统计局数据
- 第三方教育机构报告
- 招聘网站行业数据

#### 集成方式
- RESTful API 调用
- Webhook 事件监听
- 定期文件下载

## 四、数据质量保障体系

### 4.1 数据校验规则

#### 格式验证
- 字段完整性检查
- 数据类型验证
- 值域范围校验

#### 逻辑校验
- 录取线不得高于批次线
- 招生计划总数逻辑校验
- 时间序列连续性检查

#### 一致性校验
- 多源数据交叉验证
- 历史数据对比分析
- 异常值检测标记

### 4.2 质量控制流程

#### 实时质检
- 采集中间数据校验
- 异常数据自动识别
- 实时报警通知

#### 周期性核查
- 周度数据完整性检查
- 月度数据准确性审计
- 季度数据更新率评估

#### 人工审核
- 关键数据人工复核
- 异常数据专家判定
- 数据质量报告生成

## 五、错误处理与监控

### 5.1 错误分类处理

| 错误类型 | 处理方式 |
|----------|----------|
| 网络异常 | 自动重试 + 指数退避 |
| 反爬拦截 | IP切换 + 请求头轮换 |
| 数据结构变更 | 自动检测 + 人工适配 |
| 数据质量异常 | 标记隔离 + 人工审核 |

### 5.2 监控指标

#### 采集指标
- 采集成功率
- 数据更新及时性
- 异常站点统计

#### 质量指标
- 数据完整性
- 数据准确性
- 数据一致性

#### 性能指标
- 系统响应时间
- 数据处理吞吐量
- 资源使用情况

## 六、安全与合规

### 6.1 爬虫策略
- 遵守 robots.txt 协议
- 合理的请求频率控制
- 用户代理标识设置

### 6.2 数据保护
- 敏感信息脱敏处理
- 数据存储加密
- 访问权限控制

### 6.3 法律合规
- 仅采集公开信息
- 遵守数据使用协议
- 建立数据删除机制
